3.4 Fraud Detection

Using the Model as an API

Now that you’ve used a pretrained model for inference, let’s deploy that model using the RHODS model serving capabilities.

In order to do so, we need to move the model from your workbench to our S3 compatible object storage. We’ll be using the data connection you created earlier in the section Storage Data Connections. We’ll simply upload the model from a notebook.

you must have created the data connection My Storage AND added that data connection to your workbench before you can use it in a notebook. If you missed it, you can edit your workbench and add the data connection now. This will restart your workbench with the data connection attached.

  • In your Jupyter environment, open the file 2_save_model.ipynb, and follow the instructions in the notebook.

At the end of the notebook, you will have uploaded the file models/fraud/model.onnx to object storage, ready for your model server to use.
